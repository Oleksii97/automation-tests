## 🌐 Що тестуємо

**Horizontal Slider** — це HTML елемент `<input type="range">`.
Повзунок який рухається вліво/вправо і повертає числове значення.

```html
<input type="range" min="0.0" max="5.0" step="0.5" value="0">
<span id="range">0</span>  ← цифра яка показує поточне значення
```

**Реальні параметри слайдера на сайті:**
```
min  = 0.0  → мінімальне значення
max  = 5.0  → максимальне значення
step = 0.5  → крок (на скільки змінюється за один рух)
value = 0   → початкове значення
```

## 🔑 Константи на початку файлу

```js
const SLIDER  = 'input[type="range"]';
// CSS селектор для знаходження слайдера
// input[type="range"] — знайди <input> де атрибут type="range"

const DISPLAY = '#range';
// CSS селектор для елемента що показує число
// #range — знайди елемент з id="range"

const MIN  = 0;
// Мінімальне значення слайдера

const MAX  = 5.0;
// Максимальне значення слайдера

const STEP = 0.5;
// Крок — на скільки змінюється значення за один рух
```

**Навіщо виносити в константи:**
```
❌ Без константи — один рядок повторюється скрізь:
   cy.get('input[type="range"]').invoke('val', 5)
   cy.get('input[type="range"]').should('have.value', '5')
   cy.get('input[type="range"]').trigger('input')

✅ З константою — міняємо selector в одному місці:
   const SLIDER = 'input[type="range"]'
   cy.get(SLIDER).invoke('val', 5)
   cy.get(SLIDER).should('have.value', '5')
   cy.get(SLIDER).trigger('input')
```

## 🧪 Група 2 — Встановлення значення

### Основний патерн — invoke + trigger

```js
cy.get(SLIDER).invoke('val', 0.5).trigger('input').trigger('change');
// Ця комбінація з трьох команд — основний спосіб керувати слайдером

// invoke('val', 0.5)
// — встановлює value = 0.5 через jQuery
// — як написати в консолі браузера: $('input').val(0.5)
// — ТІЛЬКИ змінює HTML атрибут, JavaScript на сторінці не знає про це

// .trigger('input')
// — імітує подію input
// — спрацьовує поки юзер тягне повзунок (в процесі руху)
// — без цього деякі обробники не спрацюють

// .trigger('change')
// — імітує подію change
// — спрацьовує коли юзер відпустив повзунок (закінчив рух)
// — саме ця подія оновлює дисплей #range на цьому сайті
// — БЕЗ trigger('change') — дисплей залишається на старому значенні!
```

**Аналогія:**
```
invoke('val', 5)   — написати нове число на папері в темній кімнаті
trigger('input')   — засвітити ліхтарик — частково видно
trigger('change')  — увімкнути повне світло — всі бачать нове число ✅
```

## 🧪 Група 3 — Перевірка кроку 0.5

### Тест: всі валідні значення

```js
it('всі можливі значення від 0 до 5 з кроком 0.5', () => {
  const validValues = [];
  // Створюємо порожній масив куди будемо додавати значення

  for (let v = MIN; v <= MAX; v += STEP) {
  // for — цикл: починаємо з MIN (0), йдемо до MAX (5), крок STEP (0.5)
  // Ітерації: v = 0, 0.5, 1, 1.5, 2, 2.5, 3, 3.5, 4, 4.5, 5

    validValues.push(parseFloat(v.toFixed(1)));
    // v.toFixed(1) — округлює число до 1 знаку після коми як рядок
    //   0.1 + 0.2 в JS = 0.30000000000000004 (помилка float!)
    //   (0.3).toFixed(1) = '0.3' — правильно
    // parseFloat('0.3') — перетворює рядок назад в число 0.3
    // .push() — додає значення в кінець масиву
  }

  cy.log(`Всього валідних значень: ${validValues.length}`);
  // Виводить: "Всього валідних значень: 11"
  // 0, 0.5, 1, 1.5, 2, 2.5, 3, 3.5, 4, 4.5, 5 = 11 значень

  validValues.forEach((value) => {
  // Перебираємо кожне з 11 значень

    cy.get(SLIDER).invoke('val', value).trigger('input').trigger('change');
    // Встановлюємо кожне значення

    cy.get(SLIDER).should('have.value', `${value}`);
    // `${value}` — перетворює число в рядок (шаблонний літерал)
    // 0.5 → '0.5', 1 → '1', 2.5 → '2.5'
  });
});
```

### Тест: некратне значення округлюється

```js
it('некратне значення округлюється — 1.3 стає 1 або 1.5', () => {
  cy.get(SLIDER).invoke('val', 1.3).trigger('input');

    // expect() — починає перевірку
    // num === 1 || num === 1.5 — або 1, або 1.5 (одне з двох)
    // .to.be.true — результат виразу має бути true
  });
});
```

## ⚠️ Типові помилки при тестуванні слайдера

```
❌ Помилка 1: invoke без trigger
   .invoke('val', 5)
   → значення змінилось в HTML
   → але дисплей НЕ оновився
   ✅ Рішення: .invoke('val', 5).trigger('input').trigger('change')

❌ Помилка 2: тільки trigger('input') без trigger('change')
   .trigger('input')
   → на цьому сайті дисплей оновлює саме подія change
   ✅ Рішення: обидва trigger — input І change

❌ Помилка 3: число замість рядка в have.value
   .should('have.value', 5)    → ❌ число
   .should('have.value', '5')  → ✅ рядок

❌ Помилка 4: float помилки JS
   for(v = 0; v <= 5; v += 0.5)
   → 0.1 + 0.2 = 0.30000000000000004
   ✅ Рішення: parseFloat(v.toFixed(1))
```

## 🔍 Різниця між invoke('val') і invoke('attr')

```js
// invoke('val') — читає/встановлює ПОТОЧНЕ значення (що показує слайдер)
cy.get(SLIDER).invoke('val')          // читає: '2.5'
cy.get(SLIDER).invoke('val', 5)       // встановлює: 5

// invoke('attr', 'min') — читає HTML АТРИБУТ (що написано в коді)
cy.get(SLIDER).invoke('attr', 'min')  // читає: '0.0'
cy.get(SLIDER).invoke('attr', 'max')  // читає: '5.0'
cy.get(SLIDER).invoke('attr', 'step') // читає: '0.5'
```

**Різниця:**
