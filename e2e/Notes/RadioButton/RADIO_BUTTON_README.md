# 📻 Тести радіокнопок — повний розбір коду

> Цей документ пояснює кожен рядок файлу `radiobutton.cy.js`.  


---

## 🌐 Що тестуємо

**Сайт:** https://practice.expandtesting.com/radio-buttons

**Радіокнопка** — це елемент форми `<input type="radio">` який дозволяє вибрати
**тільки один варіант** з групи. Як перемикач каналів на старому радіо — звідси назва.

**Реальна структура сайту:**
```
#blue   → активна, вибрана за замовчуванням
#red    → активна
#yellow → активна
#black  → активна
#green  → disabled (заблокована, не можна вибрати)

⚠️ На сайті ДВІ групи радіокнопок з різними name=""
   Тому :checked завжди повертає 2 елементи (по одному з кожної групи)
```

---

## 🏗️ Структура файлу

```
describe()          — головна "папка" для всіх тестів
  beforeEach()      — виконується перед КОЖНИМ тестом
  context()         — підгрупа тестів (папка всередині папки)
    it()            — один конкретний тест
```

## 📝 Розбір коду по блоках

---

### `describe` — головний контейнер

```js
describe('Radio Button — повний набір тестів', () => {
```

**Що робить:** створює групу тестів з назвою.  
Cypress показує цю назву у звіті як заголовок.  
Всі тести файлу живуть всередині цього блоку.

---

### `beforeEach` — підготовка перед кожним тестом

```js
beforeEach(() => {
  cy.visit('https://practice.expandtesting.com/radio-buttons');
});
```

**Що робить:** відкриває сторінку **перед кожним** `it()` автоматично.

**Навіщо:** щоб кожен тест починався з чистого стану.  
Без цього стан від попереднього тесту міг би впливати на наступний.

**Аналогія:** як прибирати стіл перед кожним новим завданням —  
щоб залишки від попереднього не заважали.

---

### `context` — підгрупа тестів

```js
context('1. Стартовий стан', () => {
```

**Що робить:** групує схожі тести разом.  
Технічно `context` = `describe`, просто назва зручніша для підгруп.

## 🧪 Група 1 — Стартовий стан

### Тест: радіокнопки існують

```js
it('радіокнопки існують на сторінці', () => {
  cy.get('[type="radio"]').should('exist');
});
```

| Команда | Що робить |
|---|---|
| `cy.get('[type="radio"]')` | Знаходить всі елементи де атрибут `type` = `"radio"` |
| `.should('exist')` | Перевіряє що хоч один такий елемент є в DOM |

**Навіщо:** базова перевірка — якщо сторінка зламана і кнопок немає, цей тест впаде першим.

---

### Тест: за замовчуванням вибрана blue

```js
it('за замовчуванням вибрана тільки одна кнопка (blue)', () => {
  cy.get('#blue').should('be.checked');

  cy.get('[type="radio"]').not('#blue').not('[disabled]').each(($radio) => {
    cy.wrap($radio).should('not.be.checked');
  });
});
```

| Команда | Що робить |
|---|---|
| `cy.get('#blue')` | Знаходить елемент з `id="blue"` |
| `.should('be.checked')` | Перевіряє що кнопка вибрана |
| `.not('#blue')` | Виключає `#blue` зі списку |
| `.not('[disabled]')` | Виключає заблоковані кнопки |
| `.each(($radio) => {})` | Перебирає кожен знайдений елемент по черзі |
| `cy.wrap($radio)` | Огортає jQuery елемент в Cypress команду щоб можна було використовувати `.should()` |
| `.should('not.be.checked')` | Перевіряє що кнопка НЕ вибрана |

**Навіщо:** перевіряємо що сайт завантажується з правильним початковим станом.

---

### Тест: вибрано рівно дві кнопки

```js
it('всього вибрана рівно одна кнопка за замовчуванням', () => {
  cy.get('[type="radio"]:checked').should('have.length', 2);
});
```

| Команда | Що робить |
|---|---|
| `[type="radio"]:checked` | CSS селектор: знайди всі radio які зараз вибрані |
| `.should('have.length', 2)` | Перевіряє що знайдено рівно 2 елементи |

**Чому 2 а не 1:**  
На сайті є **дві групи** радіокнопок з різними атрибутами `name`.  
В кожній групі може бути вибрана одна кнопка → разом 2.

---

### Тест: всі активні кнопки видимі

```js
it('всі активні радіокнопки видимі і доступні', () => {
  cy.get('[type="radio"]').not('[disabled]').each(($radio) => {
    cy.wrap($radio)
      .should('be.visible')
      .and('be.enabled');
  });
});
```

| Команда | Що робить |
|---|---|
| `.not('[disabled]')` | Ігнорує заблоковані кнопки (нас цікавлять тільки активні) |
| `.should('be.visible')` | Елемент є на екрані (не прихований через `display:none`) |
| `.and('be.enabled')` | Елемент активний (немає атрибута `disabled`) |
| `.and()` | Додає ще одну перевірку в ланцюжок (те саме що `.should()`) |

## 🧪 Група 2 — Вибір радіокнопки

### Тест: вибір через .check()

```js
it('можна вибрати радіокнопку через .check()', () => {
  cy.get('#red').check().should('be.checked');
});
```

| Команда | Що робить |
|---|---|
| `.check()` | Спеціальна Cypress команда для вибору radio і checkbox |

**Чому `.check()` а не `.click()`:**
```
.click()  — просто клік на координати елемента
.check()  — розумна команда:
            ✅ працює навіть якщо кнопка прихована за label
            ✅ Cypress перевіряє що це саме checkable елемент
            ✅ не вибирає вже вибраний елемент повторно
```

---

### Тест: вибір через label

```js
it('вибір через клік на label теж працює', () => {
  cy.get('label[for="yellow"]').click();
  cy.get('#yellow').should('be.checked');
});
```

| Команда | Що робить |
|---|---|
| `label[for="yellow"]` | Знаходить `<label>` пов'язаний з input `id="yellow"` |
| `.click()` | Клік на текст label — як справжній юзер |

**Навіщо:** юзер зазвичай клікає на текст (label), а не на саму маленьку кнопку.  
Якщо `label` не пов'язаний з `input` — клік не спрацює. Цей тест це перевіряє.

---

### Тест: вибір по value

```js
it('можна вибрати кнопку по value атрибуту', () => {
  cy.get('[type="radio"]').check('black');
  cy.get('[type="radio"][value="black"]').should('be.checked');
});
```

| Команда | Що робить |
|---|---|
| `.check('black')` | Знаходить radio з `value="black"` і вибирає її |
| `[value="black"]` | Атрибутний селектор — знайди елемент де `value` = `"black"` |

**Навіщо:** зручно коли є багато кнопок і треба вибрати конкретну по значенню.

## 🧪 Група 3 — Ексклюзивний вибір

### Тест: при виборі red — blue знімається

```js
it('при виборі red — blue знімається', () => {
  cy.get('#blue').should('be.checked');
  cy.get('#red').check();
  cy.get('#red').should('be.checked');
  cy.get('#blue').should('not.be.checked');
});
```

**Що перевіряємо:** головну властивість радіокнопок —  
**не можна вибрати дві одночасно** в одній групі.

**Логіка тесту:**
```
1. Переконуємось що blue вибрана (стартовий стан)
2. Вибираємо red
3. Перевіряємо: red → вибрана ✅
4. Перевіряємо: blue → знята ✅
```

---

### Тест: в будь-який момент вибрано рівно 2

```js
it('в будь-який момент вибрана рівно одна кнопка', () => {
  cy.get('#black').check();
  cy.get('[type="radio"]:checked').should('have.length', 2);
});
```

**Навіщо:** рахуємо загальну кількість вибраних кнопок.  
Має бути рівно 2 (по одній з кожної групи) — не більше, не менше.

## 🧪 Група 4 — Задізейблена кнопка

### Тест: disabled кнопка

```js
it('задізейблена кнопка має атрибут disabled', () => {
  cy.get('#green')
    .should('be.disabled')
    .and('have.attr', 'disabled');
});
```

| Команда | Що робить |
|---|---|
| `.should('be.disabled')` | Перевіряє що кнопка заблокована |
| `.and('have.attr', 'disabled')` | Перевіряє що в HTML є атрибут `disabled=""` |

**HTML цієї кнопки:**
```html
<input type="radio" id="green" disabled="">
```

**Чому дві перевірки:**  
`be.disabled` перевіряє стан через JavaScript.  
`have.attr` перевіряє що атрибут фізично є в HTML.  
Разом — повна гарантія.

## 🧪 Група 5 — Стан після вибору

### Тест: стан змінюється

```js
it('стан змінюється при виборі іншої кнопки', () => {
  cy.get('#red').check();
  cy.get('#red').should('be.checked');

  cy.get('#yellow').check();
  cy.get('#yellow').should('be.checked');
  cy.get('#red').should('not.be.checked');
});
```

**Що перевіряємо:** після вибору нової кнопки — стара знімається.  
Це підтверджує що радіокнопки в одній групі працюють правильно.

## 🧪 Група 6 — Параметризовані тести

### Тест: перебір всіх кнопок у циклі

```js
it('кожна активна кнопка може бути вибрана', () => {
  cy.get('[type="radio"]').not('[disabled]').each(($radio, index) => {
    cy.wrap($radio).check().should('be.checked');
    cy.get('[type="radio"]:checked').should('have.length', 2);
    cy.log(`Радіокнопка #${index + 1} успішно вибрана`);
  });
});
```

| Команда | Що робить |
|---|---|
| `.each(($radio, index) => {})` | Перебирає кожен елемент. `$radio` — елемент, `index` — номер (0, 1, 2...) |
| `cy.log()` | Виводить повідомлення в лог Cypress для дебагу |
| `index + 1` | Додаємо 1 бо масиви починаються з 0, а людям зручніше бачити з 1 |

**Навіщо цей тест:**  
Замість писати окремий `it()` для кожної кнопки — один цикл перевіряє всі.  
Якщо додати нову кнопку на сайт — тест автоматично перевірить і її.

## ⚠️ Важливі нюанси цього сайту

```
1. #blue вибрана за замовчуванням — не баг, так і задумано

2. На сайті ДВІ групи радіокнопок:
   Група 1 (name="color"):  blue, red, yellow, black, green
   Група 2 (?):             своя кнопка

4. data-testid="result" — НЕ існує на цьому сайті
```
